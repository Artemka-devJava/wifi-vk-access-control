## Архитектура системы

```
WiFi Users → Proxy (8888) → AccessController → VK API
                              ↓
                           ├─ DeviceManager
                           ├─ VkSubscriptionChecker
                           ├─ VkNotificationSender
                           └─ RouterConnectionManager
                              ↓
                           SQLite Database
```

## REST API эндпоинты

### Аутентификация
- `GET /api/auth/check-access` - Проверка доступа
- `POST /api/auth/link-device` - Связывание устройства
- `GET /api/auth/user` - Информация об устройстве
- `GET /api/auth/user-by-vk` - Информация по VK ID

### Статистика
- `GET /api/stats/summary` - Общая статистика
- `GET /api/stats/devices` - Список устройств
- `GET /api/stats/access-logs` - Логи доступа
- `GET /api/stats/user` - Статистика пользователя

## Технологический стек

- **Framework**: Spring Boot 2.7.14
- **Database**: SQLite с Hibernate JPA
- **Proxy**: Netty 4.1.100
- **HTTP Client**: OkHttp3 4.11.0
- **SSH**: JSch 0.1.55
- **JSON**: Gson 2.10.1
- **Caching**: Google Guava 32.1.3
- **Logging**: SLF4J + Logback
- **Build**: Maven 3.6+
- **JDK**: Java 11+

## Особенности реализации

### 1. Кеширование подписок
- Использует Guava Cache с TTL 60 минут
- Уменьшает количество запросов к VK API
- Автоматически инвалидируется по истечении времени

### 2. Перехват трафика
- Использует iptables на роутере
- Автоматическая настройка при запуске
- Поддержка HTTP и HTTPS трафика

### 3. Управление устройствами
- Регистрация по MAC адресу
- Отслеживание времени подключения
- Связь с VK пользователем
- Логирование всех попыток доступа

### 4. Уведомления
- Отправка сообщений через VK API
- Три типа уведомлений: запрос подписки, приветствие, напоминание
- Отслеживание отправленных сообщений в БД

### 5. REST API
- Полная REST архитектура
- Глобальная обработка ошибок
- Поддержка CORS (может быть добавлена если нужна)
- Пагинация для больших наборов данных

## База данных

### Таблицы:
1. **users** - Информация об устройствах
2. **access_logs** - Логи попыток доступа
3. **notifications** - Отправленные уведомления
4. **subscription_cache** - Кеш статусов подписки (для будущего использования)
5. **settings** - Общие параметры системы

### Индексы:
- MAC адрес (уникальный индекс)
- VK User ID
- Статус подписки
- Время доступа
