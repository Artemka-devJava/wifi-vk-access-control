# WiFi VK Access Control

Сервис контроля доступа Wi-Fi на основе подписки в ВКонтакте. Приложение позволяет ограничить доступ в интернет для пользователей Wi-Fi и разрешать доступ только тем, кто подписан на определенную группу ВКонтакте.

## Описание

Этот проект реализует систему контроля доступа в Wi-Fi сеть, которая:

- **Перехватывает трафик** неавторизованных пользователей на роутере
- **Проверяет подписку** пользователя на группу ВКонтакте через API VK
- **Отправляет уведомления** в личные сообщения группы с просьбой подписаться
- **Кеширует результаты** проверок для оптимизации
- **Логирует все попытки доступа** для статистики
- **Предоставляет REST API** для управления и мониторинга

## Компоненты

### Основные компоненты:

1. **ProxyServer** - Netty HTTP прокси сервер
   - Перехватывает HTTP трафик на порту 8888
   - Обрабатывает запросы через ProxyHandler
   - Показывает Captive Portal для неавторизованных пользователей

2. **AccessController** - основной сервис проверки доступа
   - Регистрирует устройства по MAC адресу
   - Проверяет подписку через VK API
   - Логирует попытки доступа
   - Отправляет уведомления

3. **DeviceManager** - управление устройствами
   - Регистрирует и обновляет информацию об устройствах
   - Сканирует подключенные устройства
   - Получает IP адреса по MAC адресам

4. **VkSubscriptionChecker** - проверка подписки с кешированием
   - Кеширует статус подписки (по умолчанию 60 минут)
   - Уменьшает количество запросов к VK API

5. **VkNotificationSender** - отправка уведомлений в ВКонтакте
   - Отправляет запрос на подписку
   - Отправляет приветствия новым подписчикам
   - Отправляет напоминания

6. **RouterConnectionManager** - управление подключением к роутеру
   - SSH подключение к роутеру
   - Выполнение команд на роутере
   - Управление iptables правилами

7. **TrafficInterceptor** - перехват трафика
   - Установка правил iptables для перенаправления трафика
   - Управление цепями фильтрации
   - Получение списка подключенных устройств

## REST API

### Authentication & Device Management

#### Проверка доступа пользователя
```
GET /api/auth/check-access?macAddress=AA:BB:CC:DD:EE:FF&ipAddress=192.168.1.100&url=http://example.com
```

Ответ:
```json
{
  "allowed": true,
  "macAddress": "AA:BB:CC:DD:EE:FF",
  "vkUserId": 123456789,
  "isSubscribed": true,
  "userId": 1,
  "reason": "subscribed"
}
```

#### Связывание MAC адреса с VK пользователем
```
POST /api/auth/link-device?macAddress=AA:BB:CC:DD:EE:FF&vkUserId=123456789
```

Ответ:
```
Device linked successfully
```

#### Получение информации о пользователе по MAC адресу
```
GET /api/auth/user?macAddress=AA:BB:CC:DD:EE:FF
```

Ответ:
```json
{
  "id": 1,
  "macAddress": "AA:BB:CC:DD:EE:FF",
  "vkUserId": 123456789,
  "isSubscribed": true,
  "lastCheckTime": 1715000000,
  "firstSeenTime": 1714900000,
  "lastSeenTime": 1715000000,
  "deviceName": "Samsung Phone",
  "createdAt": "2024-05-05T10:30:00",
  "updatedAt": "2024-05-05T12:00:00"
}
```

#### Получение информации о пользователе по VK ID
```
GET /api/auth/user-by-vk?vkUserId=123456789
```

## Конфигурация

### application.yml

```yaml
server:
  port: 8080
  servlet:
    context-path: /api

# Конфигурация роутера
router:
  host: 192.168.1.1          # IP адрес роутера
  ssh-port: 22
  username: admin
  password: password           # Замените на реальный пароль
  wifi-interface: wlan0        # Интерфейс Wi-Fi на роутере

# Конфигурация proxy сервера
proxy:
  port: 8888
  listen-address: 0.0.0.0
  capture-host: 0.0.0.0

# Конфигурация VK API
vk-api:
  access-token: YOUR_VK_GROUP_ACCESS_TOKEN  # Access token группы
  group-id: 123456789                       # ID группы ВКонтакте
  api-version: 5.131
  base-url: https://api.vk.com/method

# Параметры проверки подписки
subscription:
  cache-ttl-minutes: 60       # Кеш статуса подписки (минуты)
  check-on-new-device: true
  notification-message: |
    Привет! Для получения полного доступа в интернет, 
    пожалуйста, подпишись на нашу группу ВКонтакте.
    После подписки перезагрузи браузер.
```

## Установка и запуск

### Требования

- Java 11+
- Maven 3.6+
- OpenWRT роутер или роутер с поддержкой SSH
- Группа в ВКонтакте с API доступом

### Шаги установки

1. **Клонируйте репозиторий:**
```bash
git clone <repo-url>
cd wifi-vk-access-control
```

2. **Получите VK API токен:**
   - Создайте группу в ВКонтакте
   - Получите токен доступа группы в разделе Settings > API usage
   - Убедитесь, что у токена есть доступ к `messages` и `groups` методам

3. **Настройте application.yml:**
   - Укажите IP адрес и учетные данные роутера
   - Введите VK API токен и ID группы
   - Настройте сетевые параметры

4. **Соберите и запустите приложение:**
```bash
mvn clean package
java -jar target/wifi-vk-access-control-1.0.0.jar
```

5. **Настройте роутер (для OpenWRT):**
   - Включите SSH доступ
   - Убедитесь, что установлены iptables
   - Приложение автоматически установит правила при запуске

## Поток работы

### 1. Новое устройство подключается к Wi-Fi:
- DeviceManager регистрирует MAC адрес в БД
- ProxyServer перехватывает первый HTTP запрос
- ProxyHandler показывает Captive Portal

### 2. Пользователь вводит VK ID:
- Отправляется POST запрос с MAC и VK ID
- AccessController связывает MAC с VK пользователем
- VkSubscriptionChecker проверяет подписку
- Если не подписан - отправляется уведомление

### 3. Пользователь подписывается:
- AccessController получает уведомление от ВКонтакте (webhook)
- Обновляется статус подписки в БД
- Доступ в интернет разрешается

### 4. Запросы трафика:
- При каждом запросе проверяется статус подписки (с кешированием)
- Если подписан - запрос пропускается
- Если не подписан - показывается страница блокировки

## База данных

### Таблица users
- Хранит информацию об устройствах
- MAC адрес как уникальный идентификатор
- Связь с VK пользователем
- Статус подписки и время проверки

### Таблица access_logs
- Логирует каждую попытку доступа
- Записывает IP адрес, запрошенный URL, причину
- Помогает анализировать использование

### Таблица notifications
- Хранит отправленные уведомления
- Тип сообщения (запрос подписки, приветствие, напоминание)
- ID сообщения в ВКонтакте для отслеживания

### Таблица subscription_cache
- Кеширует результаты проверок подписки
- Время истечения кеша
- Оптимизирует запросы к VK API

## Логирование

Логи записываются в файл `logs/wifi-access.log` с ротацией:
- Максимальный размер файла: 10MB
- Максимальное количество файлов: 10
- Уровень для приложения: DEBUG

## Безопасность

### Рекомендации:

1. **Пароли**:
   - Используйте длинный пароль для SSH подключения к роутеру
   - Хранитe конфигурацию в защищенном месте

2. **VK API**:
   - Не делитесь токеном доступа
   - Регулярно переводите его, если он скомпрометирован
   - Используйте минимально необходимые права доступа

3. **HTTPS**:
   - Рассмотрите использование HTTPS вместо HTTP
   - Установите сертификат на роутере

4. **Firewall**:
   - Ограничьте доступ к порту 8080 (API)
   - Разрешьте 8888 (proxy) только для Wi-Fi пользователей

## Решение проблем

### Проблема: "Connection refused" при подключении к роутеру
- Проверьте IP адрес роутера в конфиге
- Убедитесь, что SSH включен на роутере
- Проверьте учетные данные (username/password)

### Проблема: MAC адреса не определяются
- Убедитесь, что приложение запущено с правами администратора
- Проверьте, что arp доступен в системе
- На Windows может требоваться установка NPCAP

### Проблема: VK API ошибки
- Проверьте, что токен и ID группы правильные
- Убедитесь, что группа открыта для API
- Проверьте версию API (5.131 поддерживается)

### Проблема: Нет доступа в интернет после подписки
- Перезагрузите браузер (очистите кеш)
- Проверьте статус подписки на VK API
- Проверьте логи на ошибки
